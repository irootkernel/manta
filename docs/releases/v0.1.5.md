# Manta v0.1.5

Manta v0.1.5 is the first public release under the standalone Manta name. It includes the hardening, tag-selection, and release-readiness work completed after the local v0.1.3 baseline. Version v0.1.4 was not published and is intentionally skipped.

## Highlights

- Renamed the standalone CLI and module surfaces to Manta and removed legacy Kkachi coupling.
- Replaced single-lane rule selection with schema-v2 canonical tags and exact parser matching.
- Hardened artifact containment, symlink handling, interruption cleanup, standalone run allocation, surfaced redaction, and fail-closed extraction behavior.
- Added bounded complete-line tail extraction for oversized logs and compact deterministic evidence limits for noisy logs.
- Improved Pytest, Playwright, and fixture-backed Vitest extraction coverage.
- Bounded config, rule, and proposal inputs and hardened raw-log write and in-memory regex failures.
- Synchronized Summary and Status contracts and strengthened requirement-to-test traceability checks.

## Compatibility notes

- Config schema v1 and the removed lane-based CLI/config interfaces are not supported; migrate to schema v2 tags.
- Manta remains a standalone deterministic command runner and evidence producer. It does not plan tests, grant acceptance, or orchestrate external workflows.

## Known limitations

- Raw logs preserve original evidence and may contain unredacted values. Redaction applies only to derived surfaced artifacts.
- Rule proposals remain run-local until a maintainer explicitly reviews and promotes them.
- Platform-neutral behavior is tested across the Go suite, with stronger process-group and signal guarantees on Unix than on non-Unix platforms.
- Specialized parsers fail closed without generic fallback when their expected format does not match.
